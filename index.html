Three movements. One verdict.

A single OpenAI-compatible request resolves to fusion/<preset>, then moves through a council you can read end to end — every stage bounded, traced, and accounted for.

01

The panel, convened in parallel

The prompt fans out to several configured models at once. Each provider is explicit — type, base URL, encrypted key, and an allow-list of models it may call.

02

The judge, weighed at temperature 0

A deterministic judge reads the panel for consensus, contradictions, and likely errors — and returns structured JSON, not a vibe.

consensusdissentstructured JSON
03

The synthesis, written as one answer

A final model synthesizes the verdict in canonical OpenAI shape. The run records bounded trace metadata, usage, and cost against a single run ID.

Change the base URL. Nothing else.

Point any OpenAI client at OmniFusion and call fusion/general. Streaming, tool calls, and a minimal Responses API subset all preserve the OpenAI wire shape.

  • /v1 & /api/v1 chat completions, with canonical SSE streaming and terminal usage chunks.
  • Tool-call passthrough forwards tool-capable requests while preserving OpenAI shape.
  • openrouter/fusion alias resolves to your default fusion preset for zero-config clients.
  • Per-request plugins override panel models, synthesis model, panel cap, or web grounding.
curl http://127.0.0.1:8000/v1/chat/completions \
  -H "Authorization: Bearer $OMNIFUSION_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "fusion/general",
    "messages": [
      {"role": "user", "content": "Explain transformers, briefly."}
    ],
    "max_tokens": 512
  }'
from openai import OpenAI

client = OpenAI(
    base_url="http://127.0.0.1:8000/v1",
    api_key="your-omnifusion-client-key",
)

response = client.chat.completions.create(
    model="fusion/general",
    messages=[{"role": "user",
               "content": "Give me a launch checklist."}],
    max_tokens=500,
)

print(response.choices[0].message.content)
# canonical OpenAI-style SSE chunks + trace IDs
stream = client.chat.completions.create(
    model="fusion/general",
    messages=[{"role": "user",
               "content": "Write a haiku about SQLite."}],
    stream=True,
    stream_options={"include_usage": True},
)

for event in stream:
    if event.choices:
        print(event.choices[0].delta.content or "", end="")

Everything an endpoint needs. Nothing it shouldn’t hide.

Explicit configuration, encrypted secrets, real accounting, and bounded traces — the parts a hosted router keeps behind glass, here under your control.

Registry

Providers & presets, explicit

Register each provider’s type, base URL, key reference, and allowed models. A preset defines panel, judge, final, prompts, strategy, stage budgets, and cost ceilings — exposed as one fusion/<name>.

openai openrouter custom_openai custom_anthropic
Accounting

Budget ledger

Reserve-and-reconcile accounting in SQLite, stored as integer microdollars to avoid float drift. Global daily and per-request ceilings, with per-key concurrency caps.

Encrypted at rest

Provider keys are encrypted in SQLite with your secret key. Placeholder secrets are rejected at startup.

Web grounding

Optional search + hardened web_fetch with SSRF protection and bounded source attribution in traces.

Admin UI

Configure providers and presets, test in a playground, inspect runs, and prune invalid presets — no YAML wrangling required.

On the record

Traces & run IDs

Every stored run carries an X-OmniFusion-Run-Id header. Replay bounded stage metadata, usage, and cost through the API or admin UI whenever store is enabled.

Self-hosted

Yours to run, end to end

One container, localhost-bound by default, behind a reverse proxy you control. Bring your own provider accounts, network policy, and deployment hardening — no hosted dependency, no telemetry phone-home.

Hardened where it counts.

OmniFusion ships with defensive defaults so the safe path is the default path — not a checklist you hope someone remembers.

Bearer-auth by default

Every API request requires a token from OMNIFUSION_API_KEYS. Keys are HMAC-hashed; CORS is disabled out of the box.

Encrypted provider keys

Stored keys are encrypted with OMNIFUSION_SECRET_KEY. Treat exports and .env as secrets — they decrypt.

SSRF-guarded fetch

web_fetch allows only http/https and blocks cloud metadata, loopback, private, link-local, and reserved egress by default.

Localhost-bound

Docker Compose binds 127.0.0.1 by default. Put it behind a proxy you control and terminate TLS there.

Recursion-guarded plugins

Unknown plugin fields are rejected; plugin models must resolve to registered providers. Recursive fusion calls are blocked.

Honest about claims

No benchmark advantage is claimed from mocked tests. The experimental conductor strategy stays off until real ablation evidence clears both baselines.
